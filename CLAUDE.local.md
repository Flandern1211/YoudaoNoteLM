# === 代码红线（IMPORTANT 标注） ===
# （你从生产事故中学到的血泪教训）
- IMPORTANT: 所有数据库操作必须在 repository 层，Controller 禁止直接操作 DB
- IMPORTANT: 数据库迁移必须可逆（有 up 必须有 down），大表变更必须考虑锁
- YOU MUST: 新增 API 端点必须同时更新 OpenAPI schema
- YOU MUST: 涉及用户数据的操作必须有审计日志
- 不可以忽略任何错误，必须向上传递或者显示定义



# === 上下文压缩保护 ===
When compacting, ALWAYS preserve:
- 已修改文件列表和每个文件改了什么
- 测试运行结果
- 架构红线约束
- 当前 docs 的引用路径


# === 性能约定 ===
- 列表接口必须分页，默认 20 条，最大 100 条
- 严禁循环查询（N+1）
- 频繁读取的配置数据使用缓存，过期时间不超过 5 分钟

- 任何任务执行前都必须检查下项目中与该任务相关的部分已经更新，并询问
